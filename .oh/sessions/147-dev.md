## Phase 1: Problem Statement

### Problem
Agents must decide upfront whether to use `search_symbols` (flat search) or `graph_query` (graph traversal). These are converging -- graph_query already accepts a `query` param for semantic search, and planned changes (#141 compact, #142 batch/top_k, #145 sort_by importance) would add overlapping params to both. Two tools with different names for overlapping functionality = agent friction.

### Acceptance Criteria
- [ ] Single `search` tool registered in MCP server
- [ ] All current `search_symbols` queries work via `search`
- [ ] All current `graph_query` queries work via `search`
- [ ] `search_symbols` and `graph_query` still work as aliases (deprecated)
- [ ] `oh_search_context` remains separate (cross-domain: code + markdown + commits + artifacts)
- [ ] Default `top_k` is 10 for flat search, 1 for traversal mode
- [ ] Tests cover all param combinations in the behavior table

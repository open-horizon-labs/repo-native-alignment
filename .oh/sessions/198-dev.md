# Dev Pipeline -- #198 Stale worktree roots persist in LanceDB
**Issue:** #198
**PR:** (pending)
**Started:** 2026-03-13

## Phase 1: Problem Statement

When git worktrees are removed from disk, their symbols, embeddings, and metadata persist in LanceDB indefinitely. Every search returns 3-4x duplicate results from dead roots, inflating symbol count from ~7K to 25K.

**Root cause:** `build_full_graph` discovers live roots via `with_worktrees()` but never compares them against roots already stored in LanceDB. The background scanner has stale-root detection (`removed_slugs` diff), but this only catches removals that happen while the server is running. On restart, stale data is loaded verbatim.

**Acceptance Criteria:**
- [ ] At startup, roots stored in LanceDB but absent from disk are pruned from all tables (symbols, edges, file_index, pr_merges)
- [ ] `delete_nodes_for_roots` also cleans file_index and pr_merges (currently only handles symbols + edges)
- [ ] No data for live roots is deleted
- [ ] Tests verify stale root pruning logic

# Dev Pipeline — repo_map PageRank dominated by boilerplate
**Issue:** #190
**PR:** #196
**Started:** 2026-03-13

## Phase 1: Problem Statement

Agents using `repo_map` for cold-start orientation get actively misled. The top 15 symbols are `fmt` Display impls (boilerplate) and `Check` (a test struct), while actual architectural hubs (`RnaHandler`, `GraphIndex`, `build_full_graph`) don't appear. The importance signal conflates structural connectivity (Defines/HasField edges to fields/impls) with architectural significance.

### Acceptance Criteria
- [x] `repo_map` top 15 no longer dominated by `fmt`/`default`/`debug` impls or field nodes
- [x] Architectural hubs (high Calls-edge connectivity) rank higher than structural boilerplate
- [x] Test-only symbols (e.g., `Check` from smoke.rs) demoted
- [x] Existing PageRank tests still pass
- [x] New tests validate the improved ranking behavior

## Phase 2: Solution Space

Evaluated 4 candidates:
1. **Edge-type weighted PageRank** -- requires custom PageRank impl, petgraph doesn't support edge weights
2. **Post-filtering in repo_map** -- band-aid, doesn't fix global importance scores
3. **Weighted graph construction** -- hacky, inflates graph size
4. **Hybrid dampening + display filter** (selected) -- post-PageRank 0.1x penalty on boilerplate + Field exclusion in repo_map

Selected approach: Hybrid (candidate 4). Fixes both global importance scores and display.

## Phase 3: Execute

Implementation complete:
- `dampen_boilerplate()` method on `GraphIndex` -- applies 0.1x penalty to:
  - Functions named fmt/default/clone/eq/hash/from/into/etc. (standard trait impls)
  - Field-kind nodes
  - Test-file symbols (smoke.rs, /tests/, _test.)
- Called automatically from `compute_pagerank()` after normalization
- Field nodes filtered from repo_map display
- 6 new tests, all 656 tests passing

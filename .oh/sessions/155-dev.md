# Dev Pipeline -- Type aliases not indexed (#155)
**Issue:** #155
**PR:** #175
**Started:** 2026-03-13

## Phase 1: Problem Statement
Type aliases are invisible or use inconsistent NodeKind across languages:
- Rust: `type_item` not in RUST_CONFIG.node_kinds -- completely invisible
- TypeScript: Indexed as `NodeKind::Other("type_alias")` -- works but non-first-class
- Go: Function type aliases indexed as `NodeKind::Other("type")` -- imprecise

### Acceptance Criteria
- [x] `search_symbols("Result")` finds Rust type aliases
- [x] TypeScript type aliases use NodeKind::TypeAlias (not Other)
- [x] Go function type aliases use NodeKind::TypeAlias (not Other("type"))
- [x] TypeAlias is embeddable
- [x] Tests for each language

## Phase 2: Solution Space
Selected Option B: new `NodeKind::TypeAlias` variant. Semantically accurate, enables kind-filtered queries, and only requires changes in 5 files.

## Phase 3: Execute
Implementation complete in commit b23b5ec. All 501 tests pass.

Files changed:
- `src/graph/mod.rs` -- TypeAlias variant, Display, is_embeddable, find_node_at
- `src/extract/rust.rs` -- type_item in RUST_CONFIG + test
- `src/extract/typescript.rs` -- Other("type_alias") -> TypeAlias + test
- `src/extract/go.rs` -- Other("type") -> TypeAlias + tests
- `src/ranking.rs` -- TypeAlias tier-0 priority + test coverage

Note: tree-sitter-rust uses `type_item` (not `type_alias`) for type alias declarations. Initial attempt with `type_alias` failed; corrected after checking node-types.json.

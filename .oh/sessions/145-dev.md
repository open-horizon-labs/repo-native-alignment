## Phase 1: Problem Statement

### Issue #145: PageRank Symbol Importance

**Problem:** RNA's ranking tier 5 uses raw edge count (incoming + outgoing) as a tie-breaker, treating all connections equally. A hub symbol called by 3 highly-connected symbols matters more than a leaf called from 5 test files. Edge count misses transitivity. Agents also want explicit `sort_by: "importance"` for repo orientation.

**Acceptance Criteria:**
- [ ] PageRank computed at index time, stored in node metadata
- [ ] `sort_by: "importance"` works on `search_symbols`
- [ ] Importance score visible in `search_symbols`, `graph_query`, and `oh_search_context` output
- [ ] Persisted in LanceDB (Arrow column in both schema constructors)
- [ ] End-to-end: importance appears in actual tool output after install + rescan

### Issue #146: Repository Map

**Problem:** Agents exploring an unfamiliar codebase have no "start here" tool. All RNA tools assume you already know what you're looking for. This cold-start problem pushes agents toward Grep/Read instead.

**Acceptance Criteria (from issue):**
- Top-N symbols by importance (PageRank)
- Files with most definitions (hotspot files)
- Active outcomes and which files they touch
- Entry points (main functions, API handlers, MCP tool handlers)

## Phase 2: Solution Space

### Candidates Evaluated

**Option A: PageRank in metadata string (band-aid)**
Store PageRank as `metadata["importance"]` string. No schema change, no SCHEMA_VERSION bump. Fast to implement but violates computed-but-not-delivered metis -- string metadata isn't type-safe and requires parsing at every read site.

**Option B: Typed Arrow column + tier-5 replacement (local optimum) -- SELECTED**
1. Add `importance` Float64 column to `symbols_schema()` and `symbols_schema_with_vector()` in store.rs
2. Compute PageRank via `petgraph::algo::page_rank()` after graph build, store in `Node.metadata["importance"]`
3. Wire through all 3 layers per computed-but-not-delivered metis: extraction -> LanceDB schema -> MCP tool output
4. Replace tier-5 raw edge count with PageRank in `ranking.rs`
5. Add `sort_by: "importance"` to search_symbols (parallel to existing `sort_by: "complexity"`)
6. Add `repo_map` tool for #146 that queries top-N by PageRank + hotspot files + outcomes

**Option C: Separate importance service (redesign)**
External precomputation with caching. Overkill for in-memory petgraph with <100k nodes.

### Selected: Option B

**Rationale:** petgraph 0.8.3 includes `petgraph::algo::page_rank()` -- zero new dependencies. The computation is cheap (single pass over in-memory graph). The 3-layer delivery pattern is proven from cyclomatic complexity (#137). The repo_map tool naturally follows from having importance scores.

### Implementation Plan

1. **GraphIndex changes** (src/graph/index.rs):
   - Add `compute_pagerank()` method that calls `petgraph::algo::page_rank()` with damping=0.85, iterations=20
   - Returns `HashMap<String, f64>` mapping stable_id -> importance
   - Expose `graph()` accessor for the internal DiGraph

2. **Node metadata** (src/server.rs build_full_graph + load_graph_from_lance):
   - After `index.rebuild_from_edges()`, call `index.compute_pagerank()`
   - Write `importance` into each node's metadata
   - Persist as Float64 Arrow column in LanceDB

3. **LanceDB schema** (src/graph/store.rs):
   - Add `Field::new("importance", DataType::Float64, true)` to both schemas
   - Bump SCHEMA_VERSION to 5

4. **Ranking** (src/ranking.rs):
   - Replace tier-5 edge count with importance score from metadata

5. **Sort by importance** (src/server.rs search_symbols handler):
   - Add `sort_by: "importance"` path parallel to `sort_by: "complexity"`

6. **Render importance** in all 3 tool output sites:
   - search_symbols: show importance alongside complexity
   - format_neighbor_nodes (graph_query): show importance
   - oh_search_context include_code: show importance

7. **repo_map tool** (src/server.rs):
   - New MCP tool `repo_map`
   - Top-N symbols by importance
   - Top files by symbol count
   - Active outcomes + touched files
   - Entry points (main fns, handler fns)

# Dev Pipeline -- #156 Macro definitions not indexed
**Issue:** #156
**PR:** (pending)
**Started:** 2026-03-13

## Phase 1: Problem Statement

Agents searching for macro-defined API surface get zero results from `search_symbols`. Rust `macro_rules!` defines DSLs, builder patterns, test helpers, and derive implementations. C/C++ `#define` defines constants, function-like macros, and feature flags. None are currently extracted.

**Root cause:** Tree-sitter node types for macros are missing from language config `node_kinds` tables:
- Rust: `macro_definition` not in `RUST_CONFIG.node_kinds`
- C/C++: `preproc_def` and `preproc_function_def` not in `CPP_CONFIG.node_kinds`

**Acceptance Criteria:**
- [ ] Rust `macro_rules!` definitions indexed as Macro kind
- [ ] C/C++ `#define` macros indexed (both object-like and function-like)
- [ ] `search_symbols("my_macro")` finds the macro
- [ ] Macro is embeddable
- [ ] Tests for Rust and C/C++

## Phase 2: Solution Space

**Selected:** New `NodeKind::Macro` -- first-class, filterable, correct semantics. Macros are a distinct concept in every language that has them.

**Level:** Local optimum -- adds a new variant to the enum, updates configs, ranking. Minimal blast radius.

**Changes:**
1. `src/graph/mod.rs` -- Add `Macro` to `NodeKind`, Display ("macro"), `is_embeddable()` = true
2. `src/extract/rust.rs` -- Add `("macro_definition", NodeKind::Macro)` to RUST_CONFIG
3. `src/extract/configs.rs` -- Add `("preproc_def", NodeKind::Macro)` and `("preproc_function_def", NodeKind::Macro)` to CPP_CONFIG
4. `src/ranking.rs` -- Add `Macro` to tier 0 (primary definitions)
5. Tests for all of the above

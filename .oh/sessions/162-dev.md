# Dev Pipeline -- Generic type parameters and trait bounds not captured in metadata
**Issue:** #162
**PR:** (filled in Phase 2)
**Started:** 2026-03-13

## Phase 1: Problem Statement

RNA does not capture generic type parameters or trait bounds in symbol metadata. `struct Vec<T>` has no indication it is generic. `fn process<T: Display + Send>(item: T)` has invisible trait bounds. Agents trying to understand type compatibility cannot use this information.

### Acceptance Criteria
- [ ] Generic type parameters captured in metadata (e.g., `metadata["type_params"] = "<T: Display + Send>"`)
- [ ] Works for Rust, TypeScript, Java, Go at minimum
- [ ] Visible in `search_symbols` output when present
- [ ] Trait bounds included (not just bare type parameter names)

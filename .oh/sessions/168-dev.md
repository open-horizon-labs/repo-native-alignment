# Dev Pipeline — Special-cased nodes lack module/class Defines edges
**Issue:** #168
**PR:** #174
**Started:** 2026-03-13

## Phase 1: Problem Statement
Nodes created in `collect_js_specials()` and `collect_ts_specials()` are pushed directly to the nodes vector without emitting module-to-node `Defines` edges for imports, consts, and type aliases. Arrow functions were fixed in PR #166 but the other special-cased node types were not.

**Acceptance Criteria:**
- All special-cased nodes (imports, consts, arrow functions) have Defines edges to their parent module
- Class-scoped special nodes have Defines edges to their parent class
- Graph traversal from module node finds all symbols including special-cased ones
- Existing tests pass

## Phase 2: Solution Space
**Selected:** Add `emit_module_defines_edge()` calls for remaining special-cased node types (local optimum).
The helper already exists from PR #166. Just need to call it for consts, imports, and type aliases.

## Phase 3: Execute
Implementation complete. Added `emit_module_defines_edge()` for:
- JS/TS scalar const declarations
- JS/TS import statements
- TS type alias declarations

Added 7 new tests (3 JS, 4 TS). All 37 extractor tests pass, full suite green.

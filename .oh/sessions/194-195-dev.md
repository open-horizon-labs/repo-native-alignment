# Dev Pipeline -- Per-root graph isolation + LSP status fix
**Issue:** #194, #195
**PR:** #197
**Started:** 2026-03-13

## Phase 1: Problem Statement

### Issue #194: Per-root graph isolation, cache seeding, and global embedding cache
**Problem:** RNA treats all roots as one monolithic build unit. One dirty worktree forces a full rebuild of everything, wiping LSP edges and tripling symbol count with cross-root duplicates. With 3 agent worktrees: 7,277 -> 21,602 symbols, LSP edges non-monotonic.

**Root cause:** `build_full_graph` line 1648 uses a single `has_changes` boolean. One dirty root -> full extract + embed + graph replace for ALL roots -> LSP edges wiped.

**Acceptance Criteria (this PR):**
- [x] Per-root `has_changes` tracking -- only dirty roots re-extracted
- [x] Clean roots (including primary with LSP edges) preserved on worktree changes

**Deferred to follow-up:**
- [ ] Worktree scanner state seeded from primary root's cache (O(changed) not O(repo))
- [ ] Global content-addressed embedding cache (BLAKE3 keyed, shared across roots)
- [ ] Results default-scoped to primary root when no `root` filter specified

### Issue #195: LSP status shows "no server detected" during async warmup
**Problem:** During async warmup, status shows "LSP: no server detected" even when rust-analyzer is installed. Conflates NOT_STARTED with UNAVAILABLE.

**Acceptance Criteria:**
- [x] Four distinct LSP states visible: NOT_STARTED (hidden) / "starting..." / "pending" / "enriched (N edges)" / "no server detected"
- [x] Synchronous `which` probe at startup distinguishes "server exists" from "no server"
- [x] Status shows "LSP: pending" while enrichment is running (already works for state=1)

## Phase 2: Solution Space

### #194: Per-root rebuild isolation
**Selected:** Per-root `has_changes` with LanceDB cache loading for clean roots (Reframe level)

Candidates evaluated:
1. **Band-aid:** Just skip clean roots in extraction -- but without cache loading, clean roots would be empty
2. **Selected: Per-root + cache reuse** -- track changes per root, load clean from LanceDB, extract dirty fresh
3. **Full per-root GraphState:** Separate graph per root -- too invasive, requires query path changes
4. **Worktree cache seeding:** Seed scanner state from primary -- deferred, requires Scanner changes

### #195: LSP status fix
**Selected:** Synchronous `which` probe + SERVER_FOUND state (Local fix level)

Add state=4 (SERVER_FOUND) and `probe_for_servers()` at handler construction. Cheap PATH check, no process spawning beyond `which`.

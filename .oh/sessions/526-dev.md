## Aim

Replace the single `EXTRACTION_VERSION: u32 = 14` constant with content-addressed
per-consumer cache keys: `(blake3(event.canonical_bytes()), consumer.version())`.

## What Changed

### New: `src/extract/cache.rs`

`ConsumerCacheKey` — the canonical cache key type:
- `payload_hash: String` — blake3 hex of `event.canonical_bytes()`
- `consumer_version: u64` — from `consumer.version()`
- `ConsumerCacheKey::new(bytes, version)` — build from raw bytes
- `key_fingerprint(key)` — u64 fingerprint for storage row keys

### New: `ExtractionEvent::canonical_bytes()` in `event_bus.rs`

Stable, deterministic byte serialization of every event variant:
- Node/edge payloads sorted by stable_id before hashing (determinism)
- Kind tag prefix prevents cross-variant collisions
- Covers all 8 event variants

### New: `ExtractionConsumer::version()` in `event_bus.rs`

Default implementation returns `0`. Consumers with meaningful versioning override it.
Config-driven consumers use `blake3(config_bytes)[..8]` as `u64`.

### Updated: `CustomExtractorConsumer` in `consumers.rs`

Added `config_bytes: Vec<u8>` field. `version()` returns first 8 bytes of
`blake3(config_bytes)` as `u64`. `from_file()` constructor reads config from disk.
Changing `.oh/extractors/*.toml` auto-invalidates the cache without any manual bump.

### Updated: `EXTRACTION_VERSION` in `graph/store.rs`

Marked `#[deprecated(since = "0.1.8")]` with a note pointing to `version()`.
The value is frozen at 14 — do not bump. Historical bump log retained as reference.

The old test guards (`assert!(EXTRACTION_VERSION >= 10)` etc.) replaced by a single
`test_extraction_version_is_frozen_at_14` test that verifies the value won't drift.

### Backward compatibility

All call sites in `server/store.rs`, `server/sentinel.rs`, `server/enrichment.rs`,
`server/helpers.rs`, `server/graph.rs`, and `server/mod.rs` suppress the deprecation
warning with `#![allow(deprecated)]`. The sentinel system (`SentinelData`) continues
reading/writing `extraction_version` for compatibility with existing cache files.

## Migration notes

On first run after this lands, `ConsumerCacheKey`-based caches will all miss (new
format). Everything re-runs once, then incremental extraction is per-consumer.

Callers constructing `CustomExtractorConsumer` directly now need `config_bytes: vec![]`
(or use `CustomExtractorConsumer::from_file()`). One test fixture updated.

## Tests added

- `extract::cache` — 7 tests: equality, determinism, config-driven versioning
- `extract::event_bus` — 7 new tests: canonical_bytes determinism, kind discrimination,
  lsp_only flag, default version, version override, blake3 hash length
- `extract::consumers` — 2 new tests: version from config bytes, `from_file` constructor

## Phase 2: Wire cache into EventBus dispatch (2026-03-23)

### Changes

**`src/extract/event_bus.rs`**
- Added `HashMap<(String, ConsumerCacheKey), Vec<ExtractionEvent>>` to `EventBus`
- `emit()` and `emit_all()` changed to `&mut self` to allow cache mutation
- Added `is_cacheable() -> bool` default method to `ExtractionConsumer` trait (default: true)
  - If cacheable: check `(consumer_name, ConsumerCacheKey)` before calling `on_event`; store result after miss
  - If non-cacheable: always call `on_event` directly, no cache interaction
- Added `cache_len()` and `clear_cache()` helpers for tests
- 5 new cache hit/miss tests

**`src/extract/consumers.rs`**
- `AllEnrichmentsGate::is_cacheable() → false` — stateful across multiple events
- `PostExtractionConsumer::is_cacheable() → false` — reads filesystem
- `TreeSitterConsumer::is_cacheable() → false` — reads filesystem
- `LspConsumer::is_cacheable() → false` — external process
- `EmbeddingIndexerConsumer::is_cacheable() → false` — side effects
- `LanceDBConsumer::is_cacheable() → false` — side effects
- All `let (bus, ...)` → `let (mut bus, ...)` for `emit()` callers

**`src/extract/scan_stats.rs`**
- `ScanStatsConsumer::is_cacheable() → false` — accumulates state

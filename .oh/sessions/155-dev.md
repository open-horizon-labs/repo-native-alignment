## Phase 1: Problem Statement
Type aliases are invisible or use inconsistent NodeKind across languages:
- Rust: `type_alias` not in RUST_CONFIG.node_kinds -- completely invisible
- TypeScript: Indexed as `NodeKind::Other("type_alias")` -- works but non-first-class
- Go: Function type aliases indexed as `NodeKind::Other("type")` -- imprecise

### Acceptance Criteria
- [ ] `search_symbols("Result")` finds Rust type aliases
- [ ] TypeScript type aliases use NodeKind::TypeAlias (not Other)
- [ ] Go function type aliases use NodeKind::TypeAlias (not Other("type"))
- [ ] TypeAlias is embeddable
- [ ] Tests for each language
